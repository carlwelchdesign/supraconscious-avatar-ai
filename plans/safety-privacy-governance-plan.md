# Safety, Privacy, And Governance Plan

## Positioning
- This is reflective/spiritual journaling, not therapy, diagnosis, crisis care, medical care, or professional advice.
- The guide is inspired by Maria Olon Tsaroucha's teachings; it is not Maria and does not speak for her.
- Responses are reflective possibilities, not truth claims, predictions, supernatural certainty, or instructions.

## Guardrails
- Crisis safety is non-optional and cannot be disabled by user preference.
- High-risk entries skip council confrontation and receive grounding support.
- Pattern memory writes only when `patternMemoryEnabled` is true.
- Source material requires provenance and rights metadata before retrieval or display.
- Raw journal text remains protected in admin surfaces.

## User Controls
- Pattern memory can be disabled.
- Voice is optional and provider processing must be disclosed.
- Future controls should include entry deletion, account deletion, export, session revocation, and memory clearing.

## Admin Controls
- Raw journal reveal requires authorization, explicit reason, and audit logging.
- Source approval and prompt/config changes must be auditable.
- AI quality review should show metadata and traces before exposing raw user content.

## Safety Tests
- Self-harm and harm-to-others route to grounding.
- Abuse/coercion, severe dissociation, psychosis-like destabilization, minors, and unsafe dependency avoid symbolic confrontation.
- Red-team prompts cannot force diagnosis, certainty, channeling, or impersonation.
- Non-owner access to journal/council data is rejected.
